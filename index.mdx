description: "Migrate Work Items, Test Plans, Pipelines, and Teams between TFS and Azure DevOps projects, collections, and organizations."
---

The Azure DevOps Migration Tools let you bulk-migrate and edit data between Team Projects on Microsoft Team Foundation Server (TFS) and Azure DevOps Services. Move work items with full history, attachments, and links — within the same organization or across organizations.

<Note>
  These tools run on **Windows only** and support TFS 2013+ through the latest Azure DevOps Services. The primary command is `devopsmigration`.
</Note>

## How it works

<Steps>
  <Step title="Install the tools">
    Run `winget install nkdAgility.AzureDevOpsMigrationTools` or use Chocolatey: `choco install vsts-sync-migrator`.
  </Step>
  <Step title="Generate a configuration file">
    Run `devopsmigration init --options Basic` to create a starter `configuration.json` in your working directory.
  </Step>
  <Step title="Configure source and target">
    Edit `configuration.json` to set your source and target Azure DevOps collection URLs, project names, and Personal Access Tokens.
  </Step>
  <Step title="Add the ReflectedWorkItemId field">
    Add the `Custom.ReflectedWorkItemId` custom field to your **target** project's work item types. This tracks which items have already been migrated.
  </Step>
  <Step title="Run the migration">
    Execute `devopsmigration execute --config .\configuration.json` to start the migration. The tool validates configuration before processing begins.
  </Step>
</Steps>
